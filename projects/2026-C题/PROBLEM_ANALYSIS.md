# 赛题分析

## 1. 问题背景

某高校自研的"校园圈"社交平台存在"沉默的大多数"困境：
- 日活在开学两周后下滑，头部用户贡献大部分内容
- 信息传播存在"院系墙"、"社团墙"
- 好友推荐沿用简单"共同好友计数"逻辑，推荐同质化

## 2. 问题一：社群发现（必做）

### 目标
基于"好友关系表"和"用户属性表"，识别社交网络中的社群结构。

### 具体要求
1. 回答"有哪些社交群体"——通过社区发现算法划分社群
2. 回答"社群之间是否存在关系及关系强弱"——分析社群间连接
3. 找出**内部连接密度最大**的 **5 个社群**
4. 分析这5个社群在**成员分布、功能定位上是否存在重叠**

### 输入
- 好友关系表：S1-S200 之间的好友关系（1198 条边，无向图）
- 用户属性表：年级（大一/大二/大三/大四/研究生）、专业类别（文科/工科/理科）、具体专业、所属社团、班级

### 输出要求
- 5 个密度最大社群的成员列表
- 每个社群的特征画像（成员年级/专业/社团分布）
- 社群两两之间的重叠度分析

### 评价指标
- 社群内部连接密度（边密度或模块度）
- 社群间重叠度（Jaccard 相似度或共享成员比例）

---

## 3. 问题二：好友推荐（必做）

### 目标
基于现有数据特征，分析与 S11 还不是好友的用户中，最合适的 3 位推荐人选。

### 具体要求
1. 分析好友之间存在的**共有特性**（属性相似性、共同好友数、行为相似性等）
2. 从非 S11 好友的用户中找到最合适的 3 位
3. 分析"还不是好友"的可能原因

### 输入
- 好友关系表
- 用户属性表（年级、专业、社团、班级）
- 行为数据表（互动频率、活跃时段、话题参与度）
- 重点目标用户：**S11**（大一、文科、历史学、电子科技协会/吉他社、科技参与=2、文化参与=9、活跃时段=下午）

### 输出要求
- 3 位推荐用户及其与 S11 的相似度得分
- 相似度的计算维度和权重说明
- 未成为好友的原因分析（可能包括：无共同好友、属性差异大、不同活跃时段等）

---

## 4. 问题三：信息传播与关键用户（必做）

### 目标
构建信息传播概率模型，为"科技类话题"筛选关键用户，模拟传播过程。

### 具体要求
1. 基于数据构建信息传播概率模型
2. 筛选有利于"科技类话题"传播的关键用户
3. 模拟关键用户于**正午12:00发帖**后 **48 小时内**的传播过程

### 转发概率公式

用户 u 看到某类消息后转发概率：

$$P_{u, topic} = f(参与度_u, 互动频率_u, 首次看到时间_u)$$

公式含义（来自题面参考资料）：
- $参与度_u$：用户 u 对该类话题的参与度（行为数据表中"科技话题参与度"）
- 互动频率_u：用户 u 的平均互动频率（次/天）
- 首次看到时间_u：用户 u 首次看到帖子距发布的时间（小时）

> 活跃时段规则：
> - 上午活跃用户 → 10:00 看到
> - 下午活跃用户 → 15:00 看到
> - 晚上活跃用户 → 20:00 看到
> - 全天活跃用户 → 立即看到

> 多次看到时，每次转发事件视为独立事件。

### 输入
- 好友关系表：传播路径
- 行为数据表：参与度、互动频率、活跃时段
- 话题类型：**科技类**（对应科技话题参与度）

### 输出要求
- 关键用户排名（按传播影响力排序）
- 推荐的关键用户 ID
- 48 小时传播模拟结果（传播人数-时间曲线、传播深度、影响范围）
- 传播过程的可视化

## 5. 问题四：精准推送策略（选做）

### 目标
针对"文化类话题"，系统有每天 10 个推送名额（可分时段推送），设计推送策略使 48 小时内传播范围最大。

### 具体要求
1. 分析如何针对不同社群、不同活跃度人物进行精准推送
2. 计算最大传播范围（题目提示约 172 人）
3. 模拟验证策略

### 输入
- 问题一发现的社群结构
- 行为数据表（特别是文化话题参与度）
- 推送名额：每天 10 个，可分批次在不同时段推送
- 系统推送的用户在推送时刻**立刻看到**（不同于普通传播的延迟看到）

### 输出要求
- 推送策略（选谁、何时推）
- 模拟的传播范围
- 与题目提示值（约172人）的对比验证

## 7. 关键难点与注意事项

1. **社群发现方法选择**：图是稀疏还是稠密的？用 Louvain/Infomap/K-Means 还是谱聚类？
2. **好友推荐特征工程**：需要综合属性相似度、共同好友数、行为相似度、社团重叠度等多维特征
3. **传播模型参数确定**：转发概率公式的参数需要从数据中拟合或设定合理假设
4. **模拟效率**：48 小时、200 用户的传播模拟可能涉及多轮传播，需考虑计算效率
5. **Q4 推送优化**：组合优化问题——选择哪些用户 × 何时推送
