## 题型到模型族

| 题型 | 优先模型 | 常见辅助 | 验证方式 |
| --- | --- | --- | --- |
| 评价排序 | AHP、熵权、TOPSIS、PCA、灰色关联 | 标准化、权重扰动 | 排名稳定性、权重敏感性 |
| 预测 | ARIMA、回归、XGBoost、随机森林、LSTM、组合预测 | 变点检测、残差分析、特征工程 | RMSE、MAE、R2、残差检验 |
| 优化调度 | 线性/整数/非线性规划、遗传算法、模拟退火、NSGA-II | 情景分析、滚动时域 | 目标值、约束满足、压力测试 |
| 物理机理 | 微分方程、几何模型、动力学、数值模拟 | 参数辨识、碰撞检测 | 守恒量、极端情况、误差界 |
| 空间/区域分析 | Moran's I、LISA、K-means、热力图 | 熵权-TOPSIS、空间可视化 | 显著性、聚类轮廓系数 |
| 数据缺失/补全 | KNN、NMF、SVD、SoftImpute、PCA | TOPSIS 横评 | 交叉验证、RSE/RMSE |
| 资源配置 | 排队论、设施选址、库存演化、蒙特卡洛 | 多目标优化、CVaR | 瓶颈识别、鲁棒性 |
| 深度时空预测 | GNN、LSTM、ARIMA、分布滞后模型 | 消融实验、基线模型 | 缺口减少率、稳健性 |

## 选型原则

1. 先按子问题选模型，不要给整题套一个模型。
2. 至少给出一个可解释基线模型，再给复杂模型。
3. 如果数据少，优先可解释统计/优化模型，谨慎使用深度学习。
4. 如果题目要求策略建议，模型必须输出可解释指标，不只是预测值。
5. 每个核心模型都要配一个验证环节：残差、灵敏度、稳健性、极端情景或交叉验证。
6. 摘要中出现的模型必须在正文有公式、参数和结果支撑。

## 推荐组合模板

### 评价类

`指标体系 -> 标准化 -> 权重(AHP/熵权/PCA) -> TOPSIS/灰色关联 -> 排名稳定性`

### 预测类

`数据清洗 -> 特征/趋势分析 -> 基线模型 -> 机器学习或组合预测 -> 残差与泛化检查 -> 情景预测`

### 优化类

`变量定义 -> 目标函数 -> 约束 -> 求解器/启发式 -> 可行性检查 -> 灵敏度/压力测试`

### 物理机理类

`几何/物理假设 -> 方程建立 -> 数值求解 -> 参数估计 -> 极端情况和守恒检查 -> 可视化`

### 数据补全类

`缺失结构分析 -> 多方法补全 -> 交叉验证 -> 降维/规律挖掘 -> TOPSIS 横评 -> 稳健性`
